# Conversion Framework Reference

Loaded during Phase 3 (Structure). Use this to architect page sections, write copy, and place conversion triggers.

---

## The Hormozi Value Equation

```
Value = (Dream Outcome x Perceived Likelihood) / (Time Delay x Effort & Sacrifice)
```

Every section maps to this. Design maximizes numerator, minimizes denominator.

| Variable | Design Goal | Implementation |
|----------|-------------|----------------|
| Dream Outcome | Paint transformation vividly | Hero: gradient text headline, lifestyle imagery. Show END STATE. |
| Perceived Likelihood | Make success inevitable | Social proof, stats, progress visualizations, before/after |
| Time Delay | Compress perceived time | Speed stats, "in seconds" microcopy, progress bars, fast animations |
| Effort & Sacrifice | Make it effortless | One-click CTAs, minimal forms, "we handle it", step counters (1-2-3) |

---

## 9-Section Page Architecture (DEFAULT)

```
1. HERO          -> Dream Outcome (above fold, Z-pattern layout)
2. PAIN          -> Problem amplification (status quo is unacceptable)
3. SOLUTION      -> Your answer (with Perceived Likelihood proof)
4. VALUE STACK   -> Feature breakdown (each maps to sub-goals)
5. SOCIAL PROOF  -> Testimonials, logos, stats
6. HOW IT WORKS  -> 3-step simplicity (minimize Effort)
7. CTA           -> Single clear action
8. OBJECTIONS    -> FAQ / guarantees (reduce risk)
9. FINAL CTA     -> Last chance (urgency without manipulation)
```

## Value Stacking Formula

Each feature gets its own "value" with a price:

```
Feature 1: "AI-Powered Analysis" (Value: $497)
Feature 2: "Custom Dashboard" (Value: $297)
Feature 3: "24/7 Support" (Value: $197)
Total Value: $991
Your Price: $47/month
```

Key: Each feature links to a Dream Outcome sub-goal. Price anchoring makes the actual price feel like a steal.

---

## Price Anchoring Patterns

- Show competitor/alternative prices first (higher)
- Stack total value, then reveal actual price
- Annual vs monthly comparison (show savings)
- "Less than a coffee per day" reframing
- Crossed-out original price with current price

---

## Risk Reversal Patterns

- Money-back guarantee (30/60/90 day)
- Free trial (no credit card required)
- "Cancel anytime" prominently displayed
- Results guarantee ("or we'll work for free until you get them")
- Reversed risk: "What's the cost of NOT acting?"

## Copy Rules (Hormozi-Derived)

- One story per section. One idea. One emotion. One action.
- Headline = Dream Outcome. Subhead = How you deliver it.
- Give reasons. Every claim has evidence. "X because Y."
- Specificity > vagueness. "97% confidence" beats "high accuracy."
- Reduce friction. Remove qualifiers, hedging, passive voice.
- Max text width: 700px (45-75 chars per line).
- Active voice: "We analyze your data" not "Your data is analyzed"
- Numbers: Always specific. "247 companies" not "hundreds"
- Timeframes: "In 30 seconds" not "quickly"

## Visual Conversion Triggers

### Directional Cues — Guide the Eye to the CTA

- Arrow graphics pointing to CTA
- Images with subjects looking toward CTA
- Gradient lines flowing toward the button
- White space isolation around CTA (2rem minimum)

### CTA Design Rules

- Highest contrast element on the page
- Above fold AND repeated after every value section
- Single primary action (one button color, one verb)
- Pill shape or rounded corners (feels approachable)
- Micro-copy beneath: "Free forever. No credit card." (reduces friction)

### Social Proof Placement

- Logo strip immediately below hero (fast credibility)
- Testimonials with photos (faces build trust — 15% conversion lift)
- Specific stat counters: "10,247 teams" not "thousands of teams"
- Star ratings inline with testimonials
- Case study links for B2B

### Trust Badge Patterns

- Security badges near payment/forms
- "As seen in" media logos
- Certification logos (SOC2, GDPR, ISO)
- Number of customers/users prominently displayed
- Uptime stats for infrastructure products

---

## Urgency Without Manipulation

- Authentic scarcity: "87 spots left" (if real)
- Time-limited offers with visible countdown (if real)
- Social proof urgency: "142 people viewing this right now"
- Seasonal relevance: "Start before Q2 planning"
- NEVER fake scarcity. It destroys trust permanently.
